gsize960/siz960 Section Size Printer

The gsize960/siz960 section size printer ("sizer") prints the number of bytes required for each section of an object file, and their sum. It also displays the address for each of the sections.

The invocation command is:

gsize960 | siz960 [ -option ]... [ filenames ]

Each option must be preceded by a - character. (In Windows*, you can also use a / character.) The options are listed below. If no options are specified, sizes are displayed in decimal and addresses are displayed in hex.

filenames identifies one or more object files. If the file is an object library, the sizer reports the size for each member of the library. If file is omitted, the sizer reports the size of a.out, b.out, or e.out.

  • c
    Displays total size of COMMON symbols.
  • d
    Displays the sizes and addresses in decimal.
  • h
    Displays help information and exits.
  • n
    Includes unallocated sections in the size calculation. For example, ELF .debug_ sections are included whereas they normally would not.
  • o
    Displays the sizes and addresses in octal.
  • p
    Suppresses header display.
  • V
    Displays the section-size printer version and creation date and continues processing.
  • v960
    Displays the section-size printer version and creation date and stops processing.
  • x
    Displays the sizes and addresses in hexadecimal.
