dmp960/gdmp960Dumper/Disassembler

The dumper/disassembler provides access to binary information in executable or relocatable object files. The dmp960/gdmp960 utility combines the features of traditional disassemblers and dumpers.

The invocation command is:

dmp960 | gdmp960 [ option ]... filename...

The options (listed below) either select which part of the object file to display, or control the formatting of the display. The default selection option is d, which dumps the sections of an object file that are loaded in target memory. Each option must be preceded by a - character. (In Windows*, you can also use a / character.)

Options

  • a
    Disassembles all sections in an object file. Use this to examine the raw DWARF information in a file.
  • A{ SA | SB | KA |KB | CA | CF | JA | JD | JF | HA | HD | HT | RP | CORE0 | CORE1 | CORE2 | CORE3}
    Outputs assembly language code specific to any i960 processor type. The CORE0-3 options let you create code that is compatible with a group of processors. Here are the types of i960 processors that are supported by each CORE switch:
    CORE0 = Jx, Hx, RP
    CORE1 = All 80960 architectures
    CORE2 = Jx, Hx, RP
    CORE3 = Cx, Jx, Hx
  • c
    Displays the string table.
  • D
    Disassembles sections as hexadecimal bytes, regardless of the section type. The physical address of every fourth word appears at the beginning of each line.
  • d
    Disassembles all sections loaded into target memory. Unless otherwise specified, text sections appear as assembly language, and data sections appear as hexadecimal bytes. If no options are specified when invoking the dumper, -d is assumed as the default option.
  • e
    Applies all command line options to each member of an archive file.
  • F function
    Disassembles the specified COFF or ELF function.
  • f
    Displays the file header.
  • g[arg]…
    Dumps one or more .debug_* sections. The default argument is i (.debug_info). Use the following arguments to specify a different section, or multiple sections:
    i dump .debug_info
    l     dump .debug_line
    f     dump .debug_frame
    p     dump .debug_pubnames
    a     dump .debug_aranges
    m     dump .debug_macinfo
    A     dump all .debug_* sections
  • h
    Displays the section headers.
  • help
    Displays help information and exits.
  • i
    Display the COFF optional header.
  • l
    Displays the line numbers.
  • m
    Displays the memory map (section layout) of the file.
  • n section
    Displays the requested information for only the specified section.
  • O filename
    displays the requested information for only the specified file within an archive.
  • o
    Displays the disassembled addresses in octal. The default is hex.
  • p
    Suppresses all the header displays, producing a parseable output.
  • q
    Queries the file and display its type.
  • r
    Displays relocation entries.
  • s
    Performs symbolic disassembly. Displays labels symbolically instead of numerically whenever possible. Applies only to the d option.
  • S
    Instructs the disassembler to output symbol labels rather than their values for any symbols for which you have specified the absolute address. This option works in conjunction with the -s (lowercase) option, which instructs the disassembler to perform symbolic disassembly.
  • T
    Displays the disassembled section as assembly language, regardless of the section type.
  • t
    Displays the object file symbol-table entries, or an archive’s symbol list.
  • V
    displays the dumper version and creation date, and continues processing.
  • v960
    displays the dumper version and creation date, and stops processing.
  • x
    is ignored unless used with -t. Displays the COFF symbol-table entries as hexadecimal numbers instead of a symbolic translation of COFF information. By default, the dumper produces symbolic information.
  • z
    suppresses the translation of zeros into .word directives for text-type disassembly.
